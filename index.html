¿Qué es CentroVenezolano C.A.?

SITUACIÓN PROBLEMA

  • Proceso de nómina y registro de los ingresos y egresos en hojas de cálculo.
  • Hojas de cálculo genéricas.
  • Mayor automatización de la información.
  • Permitir la centralización y disponibilidad de la información de forma remota.

OBJETIVOS

OBJETIVO GENERAL

Desarrollar el módulo web de nómina, ingresos y egresos para la empresa Centrovenezolano C.A.

OBJETIVOS ESPECÍFICOS

  1. Evaluar los requerimientos del módulo de nómina, ingresos y egresos.
  2. Diseñar el módulo de nómina, ingresos y egresos.
  3. Desarrollar las funcionalidades del módulo de nómina, ingresos y egresos.
  4. Realizar pruebas de funcionamiento para el módulo propuesto.
  5. Implantar el módulo propuesto en la empresa.

MÉTODOS, TÉCNICAS Y/O PROCEDIMIENTOS APLICADOS

Programación extrema (XP)

  • Cambios durante el desarrollo.
  • Iteraciones.

Valores de XP

  • Desarrollo iterativo e incremental.
  • Pruebas unitarias continuas.
  • Programación en parejas.
  • Integración del equipo de programación con el cliente.
  • Simplicidad en el código.

Roles de XP

  • Programador
  • Cliente
  • Tester
  • Tracker
  • Entrenador
  • Consultor
  • Gestor

Ciclo de vida de XP

  • Fase I: Exploración
  • Fase II: Planificación de la entrega
  • Fase III: Iteraciones
  • Fase IV: Producción
  • Fase V: Mantenimiento
  • Fase VI: Muerte del proyecto

Facilidades

  • Uso de una plantilla web con componentes predefinidos.
  • Buena comunicación con el tutor industrial y tutor académico.
  • Integración con el personal administrativo de Centrovenezolano C.A. para la constante retroalimentación.
  • Existencia de las dependencias necesarias para ejecutar el sistema en estudio en el servidor.
  • Implementación del framework Javascript Sails.

Dificultades

  • Implementación del framework Javascript Sails.

Aportes

  • Se logró una mayor organización de la información referente a la nómina de los empleados.
  • El sistema permitió agilizar el proceso de cálculo de nómina.
  • Se logró centralizar los movimientos de caja chica, ingresos y egresos en un solo sistema.
  • El sistema permitió acceder a la información financiera de la empresa de manera oportuna.

Conocimientos adquiridos

  • Uso de metodologías de desarrollo ágil como XP.
  • Uso de frameworks adecuados.
  • Conceptos del patrón MVC en los frameworks de desarrollo.
  • Dominio del framework Bootstrap 3.
  • Autenticación de usuarios a través de tokens.
  • Experiencia en el desarrollo de aplicaciones web.
  • Dominio del framework AngularJS.
  • Manejo básico del framework Sails.

Conclusiones

  • Utilizar la metodología XP en el desarrollo del módulo web de nómina, ingresos y egresos permitió dar respuesta a la necesidad de un desarrollo iterativo e incremental.
  • Es evidente la utilidad de las metodologías ágiles cuando en la etapa inicial del proceso de construcción de un sistema no se tienen claros todos los requerimientos.
  • Haber implementado tecnologías como AngularJS y SailsJS permitió abarcar los requerimientos del sistema sin mayor dificultad.
  • El patrón MVC contempla la separación en: lógica de negocio, manipulación de los datos, e interacción con el usuario.

Recomendaciones

  • Si se requiere que el sistema sea implantado en la Internet se deben tomar las medidas de seguridad pertinentes.
  • Implementar autenticación a través de tokens para el manejo de las conexiones con web sockets.
  • Refinar aún más el diseño lógico de la base de datos si se requiere menor redundancia de datos.
